# Daily puzzle is seeded by each player's local calendar date, not UTC

The Daily puzzle is meant to be "the same board for every player," which reads like it implies a single global instant of change (UTC midnight). We deliberately seed board generation from each player's *local* calendar date instead: the puzzle rolls over at each player's own midnight, not simultaneously worldwide. This means two players in different timezones playing "today's" puzzle at the same real-world instant may see different boards — the "shared" framing is about low implementation cost (deterministic client-side generation, no backend) and matching how a player reasons about their own streak ("did I play today"), not strict global synchronicity. This choice is load-bearing for streak continuity logic and would be disruptive to change later (it would shift which historical days count as "played" for existing streaks), so it's recorded here rather than left as an implicit assumption.
