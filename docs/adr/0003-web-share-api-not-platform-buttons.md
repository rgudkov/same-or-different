# Daily results are shared via the Web Share API, not dedicated WhatsApp/Facebook buttons

The feature request was explicitly for WhatsApp and Facebook share buttons. We built one `navigator.share()` call (OS share sheet, which surfaces WhatsApp, Facebook, and everything else installed) with a clipboard-copy fallback for browsers without Web Share API support, instead of two bespoke platform-specific integrations. A future reader who only saw the original ask would expect two branded buttons and might "fix" this by adding them — that would be a regression, not an improvement. The reason: Facebook's `sharer.php` has not accepted pre-filled share text since ~2011 (anti-spam policy) — it only accepts a URL and pulls its own preview from that page's Open Graph tags. WhatsApp's `wa.me` link does accept pre-filled text. Building two separate buttons would therefore produce inconsistent behavior (rich result text on WhatsApp, generic link preview on Facebook) that a single "Share" button handing off to the OS doesn't have to justify — the inconsistency becomes the OS/target-app's behavior, not a visible asymmetry in our own UI.
